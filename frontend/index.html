How this works (click to expand)
  1. Input. A structured vehicle specification: the bill of materials (ECUs), vehicle configuration, the UDS diagnostic services each ECU supports, current vs. target software versions, and process standards to honour.
  2. Generate. The spec is sent to an LLM (or the offline rule-based planner if no model is configured), which plans an ordered commissioning program: open a diagnostic session, unlock security access, flash software, write parameters, validate, clear fault codes.
  3. Validate. The generated program is checked against the original spec: every ECU/UDS reference must actually exist and be supported, security access must precede any flash or write, and dependencies must resolve. The generator's output is never trusted blindly.
  4. Analyse. Cycle-time estimate, step-type breakdown, ECU coverage, and parallelisation headroom are computed to surface optimisation potential.
