How this works (click to expand)
  1. Input. A structured vehicle specification: the bill of materials (ECUs), vehicle configuration, the UDS diagnostic services each ECU supports, current vs. target software versions, and process standards to honour.
  2. Generate + self-repair. The spec is sent to an LLM (or the offline rule-based planner if no model is configured), which plans an ordered commissioning program. If it fails validation, the validator's own findings are sent back to the model for up to two corrective rounds.
  3. Validate. Every ECU/UDS reference must actually exist and be supported, security access must precede any flash or write, and dependencies must resolve. The generator's output is never trusted blindly.
  4. Estimate + optimise. Step durations come from an automatically-selected regression model (AutoML over a few candidates). A critical-path scheduler computes the unlimited-parallelism minimum (shown as a Gantt chart), and a second scheduler computes the real cycle time under a finite tester-channel count you choose.
  5. Analyse, batch, and recover. Cycle-time and coverage metrics roll up across a fleet in batch mode. A separate recovery simulator generates a validated corrective retry sub-program for a step that fails at runtime.
